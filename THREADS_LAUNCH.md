## 📌 5번 글 (한계 자백 — 신뢰 장치)

```
4/ 솔직한 한계:

▸ 한 줄 요약만 봤음. 실제 심사위원은 본문·팀·영상까지 봄
▸ 단일 패스 (같은 항목 N회 평가 안 함, 노이즈 ±1점)
▸ 비공개 풀(~37%) 보이지 않음
▸ TECH 기준은 추정 (공식 미공개)

천장이 본질적으로 낮습니다. 그래서 "정확도 측정"이 목적이에요
```

## 백래시 대응 (예상 댓글 → 답)

**Q: "내 아이디어를 함부로 평가하다니"**
A: "출처 데이터는 본인이 공개를 선택한 것입니다. 알고리즘 검증이 목적이지 작품 평가가 아닙니다. 본인 데이터 삭제 원하시면 GitHub Issue 로 요청 주세요."

**Q: "한 줄만 보고 어떻게 평가하나"**
A: "정확히 그게 알고리즘 천장을 측정하려는 이유입니다. 한 줄 요약의 한계가 어디까지인지 발표 후 정확도로 알 수 있어요."

**Q: "GPT/Claude wrapper 아니냐"**
A: "맞습니다. 핵심은 LLM이 아니라 ① 블라인딩으로 인기 편향 제거 ② 분야별 공식 기준 적용 ③ SHA256으로 사후조작 차단. 방법론 글입니다."

**Q: "모두의 창업 측 허가 받았나"**
A: "공개 데이터만 사용. 평가 대상은 알고리즘이지 심사위원이 아닙니다. 비교를 통해 양쪽 모두에게 인사이트를 주는 게 목적."
